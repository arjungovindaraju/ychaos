    HTTPRequestVerification pydantic-model ¶

    Makes requests to the specified endpoints and verifies that the service is returning a valid response along with verifying the response time of the service being less than latency field.

    The user can also specify a count attribute requesting the tool to make count number of requests to the same endpoints.

    basic_auth: Tuple[str, Union[pydantic.types.SecretStr, ychaos.testplan.common.Secret]] pydantic-field ¶

    Basic Auth authentication for the HTTP call

    bearer_token: Union[pydantic.types.SecretStr, ychaos.testplan.common.Secret] pydantic-field ¶

    Bearer token authentication for the HTTP call

    cert: Tuple[pathlib.Path, pathlib.Path] pydantic-field ¶

    The certificate to be sent for HTTP call. The tuple should contain Certificate and Key File path

    count: ConstrainedIntValue pydantic-field ¶

    Number of HTTP calls to be sent to each of the URL

    headers: Dict[str, Union[pydantic.types.SecretStr, ychaos.testplan.common.Secret]] pydantic-field ¶

    Headers to be sent with the request

    latency: ConstrainedIntValue pydantic-field ¶

    Expected Latency in ms. A latency below this value indicates successful verification

    method: str pydantic-field ¶

    HTTP method to be used

    params: Dict[str, str] pydantic-field ¶

    Query params to be sent with the request

    status_codes: List[int] pydantic-field ¶

    The list of status code which will be the comparison factor for the HTTP responses

    timeout: ConstrainedIntValue pydantic-field ¶

    Timeout in milliseconds at which the HTTP requests will timeout

    urls: List[pydantic.networks.AnyHttpUrl] pydantic-field ¶

    List of HTTP/s URLs to be requested

    verify: bool pydantic-field ¶

    Verify the target URL SSL certificates

    PythonModuleVerification pydantic-model ¶

    The Python Module verification. Runs an custom python script given the path of the script. The executable of the script is set to sys.executable by default which can be overridden by a custom executable. The script can also be passed a list of arguments that are provided in the arguments argument which are passed to the script by separating each element of the argument by space.

    The user of this plugin takes the entire responsibility of the script executed. YChaos is only responsible of running the script and collecting the exitcode of the script with which the state of the system is verified/monitored.

    arguments: List[str] pydantic-field ¶

    List of arguments to be sent to the script. The arguments in the list will be sent to the script space separated

    executable: str pydantic-field ¶

    The python shell to be used for executing the script

    path: Path pydantic-field required ¶

    The absolute path of the python script

    SDv4Verification pydantic-model ¶

    The SDV4VerificationPlugin offers the user of YChaos to configure a 3rd party Screwdriver Job that is triggered remotely by YChaos and upon the successful completion of that SDv4 job, the verification is marked as successful.

    If the SDv4 job fails/is aborted, the verification is marked as failure.

    Know more about Screwdriver CI/CD

    job_name: str pydantic-field required ¶

    Job name in the pipeline

    job_timeout: PositiveInt pydantic-field ¶

    Job Timeout in seconds

    pipeline_id: int pydantic-field required ¶

    SDv4 pipeline ID

    sd_api_token: Union[pydantic.types.SecretStr, ychaos.testplan.common.Secret] pydantic-field required ¶

    The Screwdriver pipeline/user access token to be able to start the jon in the pipeline

    sd_api_url: AnyHttpUrl pydantic-field required ¶

    SDv4 API URL

    VerificationConfig pydantic-model ¶

    The verification configuration that is executed during some state of the system to verify the system is in a favorable conditions or not.

    config: Dict[str, Any] pydantic-field required ¶

    The verification type configuration

    delay_after: float pydantic-field ¶

    delay (in ms) to be introduced after running this plugin

    delay_before: float pydantic-field ¶

    delay (in ms) to be introduced before running this plugin

    states: Union[ychaos.testplan.SystemState, List[ychaos.testplan.SystemState]] pydantic-field required ¶

    A system state or a list of system states in which this verification plugin should be executed.

    strict: bool pydantic-field ¶

    Setting this value to false implies the overall verification does not fail because of the failure of this test.

    type: VerificationType pydantic-field required ¶

    The verification type to be used.

    Defines the Type of plugin to be used for verification.
